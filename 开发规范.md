# 开发规范文档

## 一、项目模板使用规范

### 1.1 模板目录说明

项目根目录下的 `project_template/` 文件夹包含以下可复用的项目模板框架：

- **express-backend/** - Express.js 后端框架模板
- **springboot-backend/** - Spring Boot 后端框架模板
- **html-frontend/** - HTML 前端模板
- **uniapp-stencil/** - UniApp 小程序模板
- **App-framework/** - Android 应用框架模板

### 1.2 模板使用原则

#### 原则一：优先使用模板
当项目需求匹配以下技术栈时，**必须**使用对应的模板框架：

1. **后端技术选型**
   - 如果技术选型包含 `Node.js`、`Express` → 使用 `express-backend` 模板`
   - 如果技术选型包含 `Java`、`Spring Boot`、`SpringBoot` → 使用 `springboot-backend` 模板

2. **前端技术选型**
   - 如果技术选型包含 `HTML`、`原生HTML`、`传统前端` → 使用 `html-frontend` 模板
   - 如果技术选型包含 `UniApp`、`uniapp`、`小程序` → 使用 `uniapp-stencil` 模板

3. **移动端技术选型**
   - 如果技术选型包含 `Android`、`Kotlin`、`移动应用` → 使用 `App-framework` 模板

#### 原则二：模板复制规范

1. **复制位置**
   - 模板应复制到 `user_project/{user_id}/` 目录下
   - 根据项目结构，可以复制到根目录或创建子目录（如 `backend/`、`frontend/`）

2. **复制方式**
   - 使用 `shutil.copytree()` 或类似方法完整复制模板目录
   - **必须保留模板的完整目录结构**
   - **不要修改模板的核心配置文件**（如 `package.json`、`pom.xml` 等的基础配置）

3. **命名规范**
   - 如果项目包含多个模块，建议使用以下目录结构：
     ```
     user_project/{user_id}/
     ├── backend/          # 后端代码（从 express-backend 或 springboot-backend 复制）
     ├── frontend/         # 前端代码（从 html-frontend 或 uniapp-stencil 复制）
     └── 开发文档.md
     ```

### 1.3 模板定制规范

#### 允许的修改
- ✅ 修改业务逻辑代码（在模板基础上开发）
- ✅ 添加新的路由、控制器、服务
- ✅ 修改数据库配置（连接信息）
- ✅ 添加项目特定的依赖包
- ✅ 修改应用名称、描述等元信息

#### 禁止的修改
- ❌ 删除模板的核心框架文件
- ❌ 修改模板的基础架构（如中间件、配置类）
- ❌ 删除模板的通用工具函数
- ❌ 修改模板的日志、错误处理机制

### 1.4 模板功能说明

#### Express Backend 模板
- **包含功能**：MySQL 连接池、Redis 缓存、日志系统、错误处理、统一响应格式
- **配置文件**：`src/config/mysql.js`、`src/config/redis.js`
- **中间件**：`src/middleware/errorHandler.js`、`src/middleware/responseHandler.js`
- **工具类**：`src/utils/logger.js`、`src/utils/helpers.js`
- **使用方式**：在 `src/routes/` 目录下添加业务路由

#### Spring Boot Backend 模板
- **包含功能**：MyBatis Plus、Druid 连接池、Redis、统一响应格式、分页支持
- **配置文件**：`src/main/resources/application.yml`
- **通用类**：`Result.java`、`PageResult.java`
- **配置类**：`MyBatisPlusConfig.java`、`RedisConfig.java`、`WebConfig.java`
- **使用方式**：在 `src/main/java/com/example/` 下创建业务包

#### HTML Frontend 模板
- **包含功能**：Bootstrap 5、Axios HTTP 请求、工具函数库、表单验证
- **核心文件**：`assets/js/utils.js`（工具函数）、`assets/js/main.js`（主脚本）
- **使用方式**：在 `index.html` 基础上开发页面，使用 `Utils` 工具类

#### UniApp Stencil 模板
- **包含功能**：UniApp 基础配置、页面路由、全局样式
- **核心文件**：`pages.json`（页面配置）、`App.vue`（应用入口）
- **使用方式**：在 `pages/` 目录下添加新页面

#### Android App Framework 模板
- **包含功能**：Kotlin、Gradle 构建、基础 Activity
- **核心文件**：`app/src/main/java/com/example/`
- **使用方式**：在对应包下创建 Activity 和业务逻辑

## 二、通用开发规则


#### 状态码规范
- `200` - 成功
- `400` - 请求参数错误
- `401` - 未授权
- `403` - 禁止访问
- `404` - 资源不存在
- `500` - 服务器内部错误

### 2.4 错误处理规范

#### 异常处理
- **必须捕获所有异常**，不能抛出未处理的异常
- **记录错误日志**，包含错误堆栈信息
- **返回友好的错误信息**给前端，不暴露系统内部错误

#### 日志规范
- 使用模板提供的日志工具（如 `logger.js`、`Logger`）
- 日志级别：
  - `DEBUG` - 调试信息
  - `INFO` - 一般信息
  - `WARN` - 警告信息
  - `ERROR` - 错误信息

## 三、项目开发流程

### 3.1 项目初始化

1. **创建项目目录**
   - 在 `user_project/{user_id}/` 下创建项目目录

2. **复制模板**
   - 根据技术选型，从 `project_template/` 复制对应模板
   - 如果项目包含前后端，分别复制对应模板到 `backend/` 和 `frontend/`

3. **配置环境**
   - 配置数据库连接信息
   - 配置 Redis 连接信息（如需要）
   - 配置 API 地址、端口等



### 3.2 开发阶段

1. **阅读开发文档**
   - 仔细阅读 `开发文档.md`，理解项目需求
   - 按照开发文档的模块划分进行开发

2. **遵循模板规范**
   - 在模板框架基础上开发，不要重新搭建框架
   - 使用模板提供的工具类和中间件



## 四、常见问题处理

### 4.1 模板相关问题

**Q: 如何判断应该使用哪个模板？**
A: 根据项目的技术选型字段（`task_technology`）判断：
- 包含 `Node.js`、`Express` → `express-backend`
- 包含 `Java`、`Spring Boot` → `springboot-backend`
- 包含 `HTML`、`原生` → `html-frontend`
- 包含 `UniApp`、`小程序` → `uniapp-stencil`
- 包含 `Android`、`Kotlin` → `App-framework`

**Q: 项目需要多个模板怎么办？**
A: 创建子目录分别复制：
```
user_project/{user_id}/
├── backend/     # 从 express-backend 或 springboot-backend 复制
├── frontend/    # 从 html-frontend 或 uniapp-stencil 复制
└── 开发文档.md
```

**Q: 可以修改模板的核心文件吗？**
A: 可以修改业务相关配置（如数据库连接），但不要删除或修改核心框架文件（如中间件、工具类）。

### 4.2 开发相关问题

**Q: 如何添加新的 API 接口？**
A: 
- Express：在 `src/routes/` 下创建路由文件，在 `app.js` 中注册
- Spring Boot：创建 Controller 类，使用 `@RestController` 和 `@RequestMapping`

**Q: 如何处理数据库事务？**
A: 
- Express：使用模板提供的 `transaction` 方法
- Spring Boot：使用 `@Transactional` 注解


**Q: 如何使用缓存？**
A: 
- Express：使用 `src/config/redis.js` 提供的 Redis 客户端
- Spring Boot：使用 `RedisTemplate` 或 `@Cacheable` 注解

## 五、注意事项

1. **必须使用模板**：当技术选型匹配时，必须使用对应模板，不要从零开始搭建
2. **保留模板结构**：不要删除模板的核心文件和目录结构
3. **遵循命名规范**：统一使用规范的命名方式，提高代码可读性
4. **错误处理**：所有可能出错的地方都要有错误处理（可以只做「能看到报错信息、不直接白屏」这一层，不追求特别复杂的异常体系）
5. **日志记录**：关键操作和错误都要记录日志（开发环境下控制台日志即可，文件日志不是必须）
6. **代码注释**：复杂逻辑必须添加注释说明，方便同学之间互相接手代码

### 5.1 学生作业级别简化约定（重点，减少常见 Bug）

为了降低同学们在开发时踩坑的概率，这里做一些**有意的简化约定**，只要求能跑通、功能正确，不追求企业级完备性：

1. **SQL 中 LIMIT / OFFSET 的写法（强制要求）**
   - 分页相关的 `LIMIT` / `OFFSET` 不要再使用占位符：  
     **禁止：**
     ```js
     // ❌ 容易因为类型问题或驱动差异出错
     const sql = 'SELECT * FROM work_experience LIMIT ? OFFSET ?';
     const rows = await query(sql, [limitNum, offset]);
     ```
     **推荐：**
     ```js
     // ✅ 先把数字处理干净，再拼接到 SQL 里
     const safeLimit = parseInt(limitNum, 10) || 10;
     const safeOffset = parseInt(offset, 10) || 0;
     const sql = `
       SELECT * FROM work_experience
       ORDER BY sort DESC, start_date DESC
       LIMIT ${safeLimit} OFFSET ${safeOffset}
     `;
     const rows = await query(sql);
     ```
   - 关键点：**一定先用 `parseInt` 把页码、条数转成数字并做上下限控制**，再拼接到 SQL，避免类型不对导致 SQL 报错。

2. **数据库 / Redis 简化配置（本地作业环境）**
   - 为了减少「连不上库」这类问题，本项目统一推荐**本地作业环境**使用下面的默认配置（只用于课程作业，禁止用于线上环境）：
     - MySQL：
       - `host = localhost`
       - `port = 3306`
       - `user = root`
       - `password = root`
       - `database = personal_cms`
     - Redis：
       - `host = localhost`
       - `port = 6379`
       - `password = root`（如果本地没设密码，可以留空，同时把代码里的密码也留空）
   - `.env` 或配置文件里尽量写死上述默认值，同学只需要**保证本地 MySQL / Redis 按这个配置启动即可**，避免因为环境差异导致连接失败。

3. **JWT / 安全相关的简化**
   - 学生作业可以使用固定的 `JWT_SECRET`（例如：`JWT_SECRET=student-demo-secret`），不要求做复杂的密钥管理。
   - 只要能完成「登录成功 → 能访问后台接口；未登录/过期 → 不能访问后台接口」这条主线即可。

4. **Redis 可选**
   - 如果同学本机没有 Redis 或装 Redis 比较困难，可以：
     - 保留 Redis 相关代码，但在连接失败时只打日志，不影响项目启动；
     - 或者在作业中**不启用缓存逻辑**，直接从 MySQL 查数据即可。

5. **优先保证「能跑通 + 功能对」**
   - 对于学生作业，更重要的是：
     - 能启动项目（前后端都能跑起来）；
     - 主要功能（登录、增删改查）按开发文档要求能走通；
   - 性能优化、高并发、复杂部署流程可以**不做或简单带过**，避免因为过度设计引入额外 Bug。

## 六、模板快速参考

### Express Backend
- 入口文件：`src/app.js`
- 路由目录：`src/routes/`
- 配置文件：`src/config/`
- 中间件：`src/middleware/`
- 工具类：`src/utils/`

### Spring Boot Backend
- 主类：`SpringBootBackendFrameworkApplication.java`
- Controller：`src/main/java/com/example/.../controller/`
- Service：`src/main/java/com/example/.../service/`
- Mapper：`src/main/java/com/example/.../mapper/`
- 配置：`src/main/resources/application.yml`

### HTML Frontend
- 主页面：`index.html`
- 工具类：`assets/js/utils.js`
- 样式：`css/` 目录
- API 调用：使用 `axios` 或 `Utils.api`

### UniApp Stencil
- 应用入口：`App.vue`
- 页面配置：`pages.json`
- 页面目录：`pages/`
- 静态资源：`static/`

### Android App Framework
- 主 Activity：`app/src/main/java/com/example/.../MainActivity.kt`
- 资源文件：`app/src/main/res/`
- 清单文件：`app/src/main/AndroidManifest.xml`
