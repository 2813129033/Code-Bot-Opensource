# 开发规范

## 代码风格规范

### JavaScript/Vue3 规范

1. **命名规范**
   - 变量名：小驼峰命名法（camelCase）
   - 常量名：全大写下划线分隔（UPPER_SNAKE_CASE）
   - 函数名：小驼峰命名法（camelCase）
   - 组件名：大驼峰命名法（PascalCase）
   - 文件名：小驼峰命名法（camelCase）或短横线命名法（kebab-case）

2. **代码格式**
   - 使用 2 空格缩进
   - 使用单引号
   - 语句末尾使用分号
   - 对象和数组最后一个元素后不加逗号

3. **Vue3 组件规范**
   - 使用 Composition API
   - 使用 `<script setup>` 语法
   - 组件导出使用 `defineComponent`
   - Props 使用 TypeScript 类型定义

### Node.js/Express 规范

1. **项目结构**
   ```
   src/
   ├── config/         # 配置文件
   ├── controllers/    # 控制器
   ├── models/         # 数据模型
   ├── routes/         # 路由
   ├── services/       # 业务逻辑
   ├── middleware/     # 中间件
   ├── utils/          # 工具函数
   └── app.js          # 应用入口
   ```

2. **路由规范**
   - 使用 RESTful API 设计
   - 路由命名使用小写字母和短横线
   - 统一错误处理

3. **异步处理**
   - 使用 async/await
   - 统一错误捕获

### 数据库规范

1. **表命名**
   - 使用小写字母和下划线分隔（snake_case）
   - 表名使用复数形式

2. **字段命名**
   - 使用小写字母和下划线分隔（snake_case）
   - 主键统一使用 `id`
   - 时间字段统一使用 `created_at` 和 `updated_at`

3. **索引规范**
   - 为外键字段创建索引
   - 为高频查询字段创建索引
   - 复合索引遵循最左前缀原则

## Git 提交规范

### 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构代码
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具链相关

### 示例

```
feat(user): 添加用户注册功能

- 实现手机号注册
- 实现微信登录
- 添加表单验证

Closes #123
```

## API 接口规范

### 请求规范

1. **HTTP 方法**
   - GET: 获取资源
   - POST: 创建资源
   - PUT: 更新资源（完整更新）
   - PATCH: 更新资源（部分更新）
   - DELETE: 删除资源

2. **URL 规范**
   - 使用小写字母
   - 使用短横线分隔单词
   - 使用复数形式表示资源

3. **请求头**
   ```
   Content-Type: application/json
   Authorization: Bearer <token>
   ```

### 响应规范

1. **成功响应**
   ```json
   {
     "code": 200,
     "message": "success",
     "data": {},
     "pagination": {
       "page": 1,
       "page_size": 20,
       "total": 100,
       "total_pages": 5
     }
   }
   ```

2. **错误响应**
   ```json
   {
     "code": 400,
     "message": "参数错误",
     "errors": [
       {
         "field": "phone",
         "message": "手机号格式不正确"
       }
     ]
   }
   ```

### 状态码规范

- 200: 成功
- 201: 创建成功
- 400: 请求参数错误
- 401: 未授权
- 403: 禁止访问
- 404: 资源不存在
- 500: 服务器错误

## 安全规范

### 数据安全

1. **敏感数据加密**
   - 密码使用 bcrypt 加密
   - 手机号、身份证等敏感信息加密存储
   - 使用 HTTPS 传输

2. **SQL 注入防护**
   - 使用参数化查询
   - 使用 ORM 框架

3. **XSS 防护**
   - 对用户输入进行转义
   - 使用 CSP 策略

### 认证授权

1. **JWT Token**
   - Token 有效期：7天
   - 使用 RS256 算法签名
   - 存储：HttpOnly Cookie

2. **权限控制**
   - 基于角色的访问控制（RBAC）
   - 接口级别权限校验

### 前端优化

1. **资源优化**

### 后端优化

2. **缓存策略**

## 测试规范

### 单元测试

- 测试覆盖率 ≥ 80%
- 使用 Jest 框架
- 测试文件命名：`*.test.js`

### 集成测试

- 使用 Supertest 框架
